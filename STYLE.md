# 주식 데일리 문체·형식 가이드

이 문서는 사람이 읽는 규칙이자 `validate-newsletter.py`가 직접 파싱하는 코드다.
"## 금지 패턴" 아래 코드펜스는 형식을 정확히 지켜야 한다 — validator가 그 부분만
정규식으로 읽는다.

## 1. 해요체 의무

모든 문장은 해요체("~했어요", "~예요", "~해요")로 마감한다. 개조식 명사형 종결·
습니다체·반말체·속어체는 금지한다.

**좋은 예**
- "코스피가 3% 넘게 올랐어요."
- "외국인 투자자들이 순매수에 나섰어요."
- "미국 반도체주가 일제히 반등했어요."

**나쁜 예**
- "코스피가 3% 넘게 올랐다셈." (속어체 — 절대 금지, 2026-07-07~ 실제 사고 사례)
- "외국인 투자자들이 순매수에 나섰음." (개조식 명사형 종결 — 금지)
- "코스피가 상승했습니다." (습니다체 — 해요체로 통일, 혼용 금지)

## 2. 앵커·소제목 한국어 의무

모든 링크의 앵커 텍스트(클릭 가능한 글자)와 섹션 소제목은 반드시 한국어로
작성한다. 영어 기사 제목을 번역 없이 그대로 앵커 텍스트에 넣는 것을 금지한다
(2026-06-26~ 실제 사고 사례).

## 3. 영어 기사 링크 처리

영어(미국) 기사 링크는 본문에 노출하지 않는다. 미국 증시 섹션 문단에는 어떤
링크도 넣지 않는다. 하단 "📎 참고 링크" 섹션에만 한글 번역 제목 + 출처명 +
원문 링크로 모아서 제공한다. 한국 기사 링크는 지금처럼 국내 증시 섹션 본문에
인라인으로 유지한다.

## 4. 링크 URL 규칙

URL은 수집 단계에서 확보한 실제 기사 URL만 사용한다. 한 글자도 변형하지 않고
그대로 복사한다. 없는 URL은 절대 만들거나 추측하지 않는다.

## 금지 패턴
```regex
다\s?셈
하셈
라\s?셈
해보셈
확인 불가
```

위 목록은 2026-07-07~ 매크로 데일리에서 실제로 발생한 속어체 유출 사고의
재발 방지용 시드 패턴이다. 새로운 사고 사례가 발견되면 이 코드펜스 안에
정규식을 추가한다.
